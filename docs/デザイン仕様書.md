# デザイン仕様書：病棟シフト作成アプリ

作成日: 2026-04-07
デザイン方針: モダン・クリーン（shadcn/ui ベース）

---

## 1. デザインコンセプト

**「迷わず使える、信頼できるツール」**

- 余白を広くとり、情報を整理して表示
- shadcn/ui のデフォルトスタイルをベースに統一感を確保
- 派手なアニメーションは使わず、操作への反応は明確に
- 20〜30人分のデータを見渡せる視認性を重視

## 4. 画面構成（ページ一覧）


### 5-2. シフト表（/schedule）← メイン画面

```
┌─────────────────────────────────────────────────────┐
│ ヘッダー                                              │
├──────────────────────────────────────────────────────┤
│ ◀ 2026年4月 ▶  [シフトを生成する]  [PDF出力]  [公開] │
├────────┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┤
│ 氏名    │1 │2 │3 │4 │5 │6 │7 │8 │9 │…│29│30│31│
├────────┼──┼──┼──┼──┼──┼──┼──┼──┼──┼──┼──┼──┤
│ 山田 太郎│日│日│夜│明│休│日│日│夜│明│…│  │  │  │
│ 鈴木 花子│早│早│日│日│夜│明│休│早│早│…│  │  │  │
│ …      │  │  │  │  │  │  │  │  │  │  │  │  │  │
├────────┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┤
│ 配置数確認バー（最低人数 充足 / 不足を色で表示）        │
└──────────────────────────────────────────────────────┘
```

**インタラクション**
- セルクリック → シフト種別を選択するポップオーバーが開く（Phase 2: ドラッグ&ドロップ）
- 不足セルは赤くハイライト
- 行ホバーでスタッフの月間集計（勤務日数・夜勤数）をツールチップ表示

### 5-3. シフト自動生成（/schedule/generate）

```
┌──────────────────────────────────────┐
│ シフト自動生成                          │
│                                       │
│ 対象月: [2026年 4月 ▼]               │
│                                       │
│ ✅ スタッフ情報      登録済み 28名      │
│ ✅ シフト種別        早番/日勤/遅番/夜勤 │
│ ✅ 希望休申請        承認済み 12件      │
│ ✅ 勤務制約          設定済み           │
│ ✅ ペア制約          設定済み 3件       │
│                                       │
│ ┌──────────────────────────────────┐ │
│ │    🔄 シフトを自動生成する          │ │
│ └──────────────────────────────────┘ │
│                                       │
│ ⚠️ 生成後は下書きとして保存されます。   │
│   内容確認後「公開」ボタンで確定します。  │
└──────────────────────────────────────┘
```

**生成中の状態**
- ボタンがローディングスピナーに変わる
- 「最適なシフトを計算中... (OR-Tools)」のメッセージ表示
- 完了後にシフト表画面へ遷移

## 6. コンポーネント方針

| コンポーネント | 使用ライブラリ |
|-------------|-------------|
| ボタン・フォーム・ダイアログ | shadcn/ui |
| シフト表グリッド | 独自実装（React-Big-Calendar または CSS Grid） |
| テーブル | shadcn/ui DataTable（TanStack Table） |
| トースト通知 | shadcn/ui Sonner |
| ローディング | shadcn/ui Skeleton |
| アイコン | lucide-react |

---

## 7. レスポンシブ方針

| デバイス | 対応方針 |
|---------|---------|
| PC（1280px以上） | フル表示 |
| タブレット（768px〜） | サイドバー折りたたみ・シフト表横スクロール |
| スマホ（〜767px） | 管理機能のみ対応（シフト表は表示のみ） |

---

## 8. アクセシビリティ

- WCAG 2.1 AA 準拠目標
- コントラスト比 4.5:1 以上
- キーボード操作対応（Tab / Enter / Arrow）
- シフト表のカラーには必ずテキスト略称を併記（色覚多様性対応）
