# 要件定義書：病棟シフト作成アプリ

作成日: 2026-04-06  
プロジェクトNo: 12  
リポジトリ: shift-create-app  
案件種別: 受注案件

---

## 1. プロジェクト概要

病棟（医療・看護）向けのシフト表作成Webアプリ。
スタッフ情報と各種条件を登録し、ワンクリックで1ヶ月分のシフトを自動生成する。

---

## 2. ターゲット・規模

- 対象: 病棟の師長・管理者（シフト作成担当者）のみ　※スタッフはアプリを操作しない
- スタッフ規模: 20〜30人
- 運用: 師長がスタッフから希望休を口頭・紙で収集し、管理者として代理入力する

---

## 3. 最優先機能

**自動スケジューリング**  
条件を入力してボタンを押すと、1ヶ月分のシフトが自動生成される。

### 自動生成の入力条件

| # | 条件 | 内容 |
|---|------|------|
| a | スタッフ情報 | 氏名・資格・雇用形態・勤務可能時間 |
| b | シフト種別 | 早番・日勤・遅番・夜勤などの時間帯・色設定 |
| c | 希望休・有給申請 | スタッフが事前に申告した休暇希望 |
| d | 最低配置人数 | 時間帯ごとの必要最低人数 |
| e | 連続勤務上限 | 連続して勤務できる日数の上限 |
| f | 明け自動挿入 | 夜勤翌日に明けを自動で配置 |
| g | スキルバランス | 新人のみの夜勤を防ぐなどの配置制約 |
| h | ペア制約 | 特定スタッフを必ず同じシフトに入れる／入れない指定（例: A夜勤時はBとペア、CとDは同シフト禁止） |

---

## 4. 機能要件

### Phase 1（〜2026-05-01）※最優先
- [ ] スタッフ登録・編集・削除（資格・雇用形態・勤務可能時間含む）
- [ ] シフト種別の登録（名称・時間帯・色）
- [ ] 希望休・有給申請の入力
- [ ] 最低配置人数・勤務制約の設定
- [ ] **自動スケジューリング**（条件入力 → ワンクリック生成）
- [ ] 月次シフト表の表示（グリッドビュー）
- [ ] 認証（管理者のみ・シングルロール）

### Phase 2
- [ ] ドラッグ&ドロップでのシフト手動編集
- [ ] PDF出力・印刷対応

### Phase 3
- [ ] 実績勤務時間の自動集計
- [ ] 残業・時間外集計（36協定対応）
- [ ] 月次レポート・CSV出力
- [ ] Supabase Realtimeによるリアルタイム通知
- [ ] スケジューリング最適化アルゴリズムの改善

---

## 5. 非機能要件

- **セキュリティ**: HTTPS必須・Supabase Auth（管理者認証）・監査ログ
- **個人情報**: スタッフ個人情報の暗号化保存
- **パフォーマンス**: 30名 × 1ヶ月分のシフトを快適に表示・生成
- **アクセシビリティ**: WCAG 2.1 AA準拠目標
- **対応デバイス**: PC・タブレット対応

---

## 6. 技術スタック

| 種別 | 採用技術 |
|------|---------|
| フロントエンド | Next.js 14（App Router）/ TypeScript / Tailwind CSS |
| UIコンポーネント | shadcn/ui |
| カレンダー | React-Big-Calendar + date-fns |
| バックエンド（メイン） | Next.js API Routes |
| バックエンド（スケジューリング） | Python + OR-Tools（Google CP-SAT） |
| DB・認証 | Supabase（PostgreSQL + Auth + Realtime） |
| デプロイ（フロント） | Vercel |
| デプロイ（Pythonサービス） | Google Cloud Run |
| コンテナ | Docker |
| メール通知 | Resend（Phase 2） |

### アーキテクチャ

```
ブラウザ（Next.js）
  ↓ シフト生成リクエスト
Next.js API Routes（Vercel）
  ↓ HTTP POST
Python スケジューリングサービス（Cloud Run）
  └── OR-Tools CP-SAT ソルバー
  ↓ 生成結果を返す
Next.js API Routes → Supabase に保存
  ↓
ブラウザにシフト表を表示
```
