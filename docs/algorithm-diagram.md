# シフト生成アルゴリズム フロー図

> `shift-rules.md` のアルゴリズム（Step A〜F）を図示したものです。

---

## 用語説明

| 図中の表現 | 意味 |
|-----------|------|
| フリーズ | 以降のステップで上書き不可にすること |
| 過剰スタッフ | 実績休日数が目標を超えているスタッフ |
| 不足スタッフ | 実績休日数が目標に届いていないスタッフ |
| 定休日のみ休日スタッフ | スタッフ設定で「定休日以外には公休を入れない」に設定されている人 |
| シニア | 師長・主任（役職）を指す |
| カバレッジ | 平日に必ずシニアが1人以上日勤に入っている状態 |

---

## 全体フロー

```mermaid
flowchart TD
    START([シフト生成開始])
    START --> A1

    subgraph SA["▶ Step A　事前確定（フリーズ）"]
        A1["A-1　前月持ち越し\n前月末の夜勤者 → 1日目を明け・翌々日を公 でフリーズ"]
        A2["A-2　休暇リクエスト\n有給・希望休・特別休暇 をフリーズ"]
        A3["A-3　ハード定休日\n「厳守」設定の定休曜日・祝日 を 公 でフリーズ"]
        A4["A-4　ソフト定休日\nフリーズしない\n定休日のみ休日スタッフ には 定休日以外の公休を禁止する制約を追加"]
        A1 --> A2 --> A3 --> A4
    end

    A4 --> B1

    subgraph SB["▶ Step B　夜勤割り当て"]
        B1["全日程 × 全スタッフを順に確認\n夜勤に入れるかチェック（詳細は下図）"]
        B2["夜・明・公 を3日分フリーズ"]
        B1 -->|"条件OK"| B2
        B1 -->|"条件NG"| B3[スキップ]
    end

    B2 --> C1
    B3 --> C1

    subgraph SC["▶ Step C　シニア日勤確保（平日のみ）"]
        C1{"平日なのに\nシニアが日勤ゼロ？"}
        C2["シニアを日勤に強制割り当て"]
        C1 -->|"Yes"| C2
        C1 -->|"No"| C3[スキップ]
    end

    C2 --> D1
    C3 --> D1

    subgraph SD["▶ Step D　一般日勤で最低人数を充足"]
        D1["候補を優先度順にソート\n① ハード定休スタッフ優先\n② シニアは後回し\n③ 公休が多い順\n④ 日勤が少ない順"]
        D2["最低人数に達するまで日勤に割り当て"]
        D1 --> D2
    end

    D2 --> E0

    subgraph SE["▶ Step E　休日数調整"]
        E0["E-0　定休日のみ休日スタッフ の調整\n定休日以外の公（フリーズ外）を 日 に変換"]
        ES["E-swap　過剰↔不足スタッフのスワップ\n同じ日に 公（過剰） と 日（不足） を交換\n日勤人数を変えないまま休日数を均す"]
        E1["E-1　過剰スタッフの 公→日 変換\n休日が多すぎるスタッフの公を日に変換"]
        E2["E-2　不足スタッフの 日→公 変換\n日勤人数に余裕がある日のみ変換"]
        E2B["E-2b　シニア専用スワップ\n非シニアの公 と シニアの日 を交換してシニアの休日を確保"]
        E0 --> ES --> E1 --> E2 --> E2B
    end

    E2B --> F1

    subgraph SF["▶ Step F　翌月繰越計算"]
        F1["不足日数を計算\n目標休日数 − 実績休日数"]
        F2["不足分を翌月の目標に上乗せ"]
        F1 --> F2
    end

    F2 --> END([シフト生成完了])
```

## Step E 詳細フロー

```mermaid
flowchart TD
    IN([Step E 開始]) --> E0

    E0["E-0　定休日のみ休日スタッフ の調整\n定休日以外にある フリーズ外の公 → 日 に強制変換\n理由：このスタッフは定休日以外は日勤に入る設定のため"]
    E0 --> ES

    ES{"同じ日に\n休日が多すぎる人の公 と\n休日が足りない人の日 がある？"}
    ES -->|"Yes"| ES2{"スワップ可能？\n① 多すぎる人が日勤に入れるか\n② ペア制約に違反しないか\n③ シニアカバレッジが維持されるか"}
    ES2 -->|"OK"| ES3["公 ↔ 日 をスワップ\n日勤人数は変わらない"]
    ES2 -->|"NG"| E1
    ES3 --> E1
    ES -->|"No"| E1

    E1{"休日が多すぎるスタッフがいる？"}
    E1 -->|"Yes"| E1B{"週末・連勤超過・\nペア制約違反・土日上限超過\nのいずれかに該当？"}
    E1B -->|"Yes"| E2
    E1B -->|"No"| E1C["公 → 日 に変換\n※シニアカバレッジチェックは現状なし（既知の課題）"]
    E1C --> E2
    E1 -->|"No"| E2

    E2{"休日が足りないスタッフがいる？"}
    E2 -->|"Yes"| E2CHK{"その日の日勤人数が\n最低ラインぴったり？\n（余裕なし）"}
    E2CHK -->|"Yes"| E2BB
    E2CHK -->|"No（余裕あり）"| E2C{"シニアスタッフ？"}
    E2C -->|"Yes"| E2D{"他のシニアが\n日勤に入れる？\nカバレッジを確認"}
    E2D -->|"Yes"| E2E["日 → 公 に変換"]
    E2D -->|"No"| E2BB
    E2C -->|"No"| E2E
    E2E --> E2BB
    E2 -->|"No"| E2BB

    E2BB{"シニアの休日が\nまだ足りない？"}
    E2BB -->|"Yes"| E2BC{"非シニアの公 と シニアの日 を\n交換できる？\nカバレッジが維持されるか確認"}
    E2BC -->|"OK"| E2BD["非シニア:公→日\nシニア:日→公 をスワップ"]
    E2BC -->|"NG"| E2BE["変換を取り消し（ロールバック）"]
    E2BB -->|"No"| OUT

    E2BD --> OUT
    E2BE --> OUT

    OUT([Step E 完了])
```

## 目標休日数の算出と繰越

```mermaid
flowchart TD
    A["前月からの繰越日数"] --> SUM
    B["当月の土日祝の日数\n（月ごとの基準日数）"] --> SUM
    C["有給の取得日数\n※有給のみ目標に上乗せ"] --> SUM
    SUM(["目標休日数"])

    NOTE["希望休・特別休暇は\n公休を日付指定したもの\n→ 基準日数の枠内でカウント\n→ 目標日数は増えない"]

    SUM --> CALC["残り公休の割当数\n＝ 目標 − 確定済み休日数\n（希望休・有給・特別休暇）"]
    NOTE -.->|"参考"| CALC

    CALC --> CMP{"当月の実績休日数\nと比較"}
    CMP -->|"実績 ≧ 目標"| OK["目標達成"]
    CMP -->|"実績 ＜ 目標"| NG["不足分を翌月の目標に上乗せ"]
```
