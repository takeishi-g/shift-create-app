# シフト制約定義書

> このファイルはシフト自動作成の「正しい動作」を業務ルールとして定義したものです。
> コード修正の際は必ずこのファイルを参照し、制約を破らないようにしてください。
> アルゴリズムの実装詳細は `shift-rules.md` を参照してください。

---

## 1. シニアペア制約（最高優先度）

**対象**: 師長・主任（シニアスタッフ2名）

**ルール**:
- 平日（土日・祝日を除く）は、師長・主任のどちらか1人以上が必ず日勤に入ること
- どちらかが夜勤・明け・公休・有給・希望休のいずれかの場合、もう片方は必ず日勤になること

**土日祝日**: 制約なし

**優先度**: 他のすべての制約との競合時にこの制約が勝つ

---

## 2. 一般ペア制約

スタッフ管理画面で任意のスタッフ2名に設定できる制約。

### 同勤務強制（must_pair）

- **目的**: 新人と教育係を必ず同じシフトに割り当てる
- **ルール**: 指定した2名は同じ日に同じシフト種別に入ること

### 同勤務禁止（must_not_pair）

- **目的**: 準看護師同士の夜勤を禁止するなど
- **ルール**: 指定した2名は同じ日に同じシフト種別に入らないこと

---

## 3. 日勤制約

### 平日

| 設定項目 | 内容 |
|---------|------|
| 最低日勤人数 | 勤務制約設定画面で設定（デフォルト: 5人） |
| シニア要件 | シニアが1人以上日勤に入ること（シニアペア制約による） |

### 土日祝日

| 設定項目 | 内容 |
|---------|------|
| 最低日勤人数 | 勤務制約設定画面で設定（デフォルト: 3人） |
| 最大日勤人数 | 勤務制約設定画面で設定（デフォルト: 3人） |
| シニア要件 | なし |

### 日勤専従スタッフ

夜勤回数（`max_night_shifts`）を `0` に設定することで夜勤不可とする。

### シニアスタッフの夜勤回数

師長・主任の夜勤回数上限に役職による特別な制限はない。個人設定の `max_night_shifts` に従う。

## 4. 夜勤制約

### 基本パターン

```
夜 → 明 → 休
```

### 夜勤回数が多いスタッフのパターン

```
夜 → 明 → 夜 → 明 → 休 → 休
```

### ルール一覧

| ルール | 内容 |
|--------|------|
| 連続夜勤上限 | 最大2連続まで |
| 夜勤翌日 | 必ず「明」（フリーズ） |
| 夜勤翌々日 | 原則「公休」（フリーズ）。ただし連続夜勤時は除く（下記参照） |
| 通常パターン | 夜→明→公（夜勤1回につき翌日明け・翌々日公休） |
| 連続夜勤パターン | 夜→明→夜→明→公→公（夜勤回数が多いスタッフに適用）。2連続夜勤後に2日間休日 |
| 月跨ぎ夜勤 | 月末に夜勤を組んだ場合、翌月1日が「明」・翌々日が「公」として確定（連続夜勤の場合は明→夜→明→公→公） |

## 5. 公休ルール

### 公休（`公`）の種類

シフト表上はすべて同じ `公` として表示されるが、アルゴリズム内部では以下の2種類を区別する。

| 種類 | 発生条件 | 上書き |
|------|---------|--------|
| 夜勤後公休 | 夜勤の翌々日に自動確定（フリーズ） | 不可 |
| 通常公休 | 目標休日数を達成するためにアルゴリズムが配置 | 可（日勤との交換対象） |

業務上（シフト表の見た目）は区別不要。どちらも公休として扱う。

### 月間公休日数

- **基準**: その月の土日祝日の数
- **設定**: 勤務制約設定画面で変更可能
- **不足時**: 翌月に繰越（必須取得）

### 目標休日数の計算

目標休日数 ＝ 月間公休日数基準 ＋ 繰越日数

※「目標」と表記するが、不足分は翌月繰越となるため実質的に必須取得。

**カウント対象**: `公`・`有`・`希休`・`他` の4種類をすべて合算して目標と比較する。
`希休`（希望休）・`他`（特別休暇）はフリーズ済みのため、アルゴリズムは残り枠（目標 − 既取得分）を `公` で補完する。

**例**: 基準8日・繰越2日・希望休1日の場合 → 目標10日、残り `公` 割当は 10 − 1 ＝ 9日

### 休日の種類

| 種別 | 公休カウント | 備考 |
|------|------------|------|
| 公休（`公`） | ○ | アルゴリズムが自動割当 |
| 有給（`有`） | ○ | 別枠・フリーズ |
| 希望休（`希休`） | ○（公休に含む） | フリーズ |
| 特別休暇（`他`） | ○（公休に含む） | 別枠・フリーズ（慶弔・研修等） |
| 夜勤明け（`明`） | × | 夜勤とセットのシフト。公休カウント外・日勤不可 |
| 定休日 | ○ | 個人設定・ソフト/ハード選択可 |

### 定休日

- **設定場所**: スタッフ管理画面（個人ごとに設定）
- **強度選択**:
  - `ハード`: 厳守。アルゴリズムで上書き不可
  - `ソフト`: 可能な範囲で守る。人数不足時は例外あり
- **祝日の定休扱い**: 個人設定で有効/無効を選択可能

#### 夜勤ありスタッフの定休日活用

夜勤が設定されているスタッフは、定休日の2日前に夜勤を割り当てることを優先する。

```
定休日 X の場合:
  X-2: 夜（夜勤）
  X-1: 明（夜勤明け）
  X  : 休（夜勤後の必須公休 = 定休日）← 重なる
```

- 夜勤後の必須公休（`post_night_off`）が定休日と重なるため、余分な公休日が増えない
- ハード定休日・ソフト定休日の両方に適用
- **2パス方式で実装**: 師長・主任の夜勤を先に確定する専用 LP（pass 1）を実行し、その結果でシニアシフトを固定してから全スタッフ LP（pass 2）を解く。これにより `senior_day_coverage` を先に満たした状態で夜勤誘導が有効に機能する
- 実装: 全スタッフ対象、CSP の目的関数で `n[X-2]` に報酬係数 `-3` を付与。pass 1 が失敗した場合は従来のシングルパスにフォールバック

## 6. 風呂日

- **概要**: 患者の入浴補助のため、追加スタッフが必要な日
- **設定場所**: 制約設定画面
- **設定内容**: 風呂日の曜日・最低必要人数（デフォルト: 平日日勤最低人数と同じ。設定画面で変更可能）
- **ルール**: 指定曜日の日勤人数が風呂日の最低人数以上であること

---

## 7. 制約の優先度

| 優先度 | 制約 | 強度 |
|--------|------|------|
| 1（最高） | シニアペア制約（平日シニア日勤確保） | ハード |
| 2 | フリーズセル（有給・希望休・夜勤・明け・ハード定休日） | ハード |
| 3 | 夜勤連続上限（最大2連続） | ハード |
| 4 | 日勤最低人数（平日・土日別） | ハード |
| 5 | 風呂日最低人数 | ハード |
| 6 | 一般ペア制約 | ハード |
| 7 | 月間公休日数の達成 | ハード（±1〜2日の誤差あり） |
| 8 | ソフト定休日 | ソフト（可能な範囲で遵守） |

## 8. シフト管理者の手動作成（参考手順）

管理者が手動でシフトを組む際の実際の進め方。アルゴリズムの設計参考でもある。

1. 前月末の夜勤者を確認し、1日の「明」と翌々日の「公」を先に確定させる（月跨ぎ夜勤の処理。連続夜勤の場合は明→夜→明→公→公）
2. 有給・希望休を記入する
3. 師長・主任の夜勤日を決める（平日のシニアカバレッジを先読みして確認）
4. 一般スタッフの夜勤を割り当てる（「夜→明→休」のブロック単位で考える）
5. 日勤で最低人数を充足させる
6. 残りのスタッフに公休を割り当てる（月間目標日数に合わせる）
7. 風呂日の日勤人数を確認・調整する
8. 全体を見直してペア制約・定休日の遵守を確認する
