# デザイン仕様書：病棟シフト作成アプリ

作成日: 2026-04-07
デザイン方針: モダン・クリーン・華やか（shadcn/ui ベース / ローズ×ラベンダー系）

---

## 1. デザインコンセプト

**「迷わず使える、信頼できるツール」**

- 余白を広くとり、情報を整理して表示
- shadcn/ui のデフォルトスタイルをベースに統一感を確保
- ローズ×ラベンダーの華やかな配色で、女性ユーザーが使いやすい雰囲気に
- 派手なアニメーションは使わず、操作への反応は明確に
- 20〜30人分のデータを見渡せる視認性を重視
- シフト表は「夜勤・遅番のみ色付き、それ以外は控えめ」で情報過多を避ける

## 5. 画面別UI仕様

### 5-1. ダッシュボード（/dashboard）

```
┌─────────────────────────────────────────┐
│ ヘッダー: 日時 / ユーザーメニュー              │
├──────────┬──────────┬───────────────────┤
│ 今月の     │ 登録済み  │  今月のシフト充足率  │
│ スタッフ数  │ 希望休数  │  ████████░░ 80%  │
├──────────┴──────────┴───────────────────┤
│ 直近のアクション（タイムライン）               │
│ - Aさんが希望休を申請（4/15）               │
│ - 4月シフトが自動生成されました               │
├─────────────────────────────────────────┤
│ クイックアクション                           │
│ [シフトを生成する]  [希望休を確認]  [スタッフ管理] │
├─────────────────────────────────────────┤
│ 備考欄（希望休登録時の備考メモ一覧）            │
│ 山田 太郎  4/15  「母の通院のため」           │
│ 鈴木 花子  4/20  「私用のため終日不可」        │
└─────────────────────────────────────────┘
```

**備考欄の仕様**
- 希望休・有給申請時に入力された備考メモを日付降順で表示
- 表示項目: スタッフ名 / 対象日 / 備考内容
- 未読の備考は強調表示（左ボーダー Rose-500）

```
│ 鈴木 花子│早│早│日│日│夜│明│休│早│早│…│  │  │  │
```

```
┌──────────────────────────────────────┐
│ シフト自動生成                          │
│                                       │
│ 対象月: [2026年 4月 ▼]               │
│                                       │
│ ✅ スタッフ情報      登録済み 28名      │
│ ✅ シフト種別        早番/日勤/遅番/夜勤 │
│ ✅ 希望休申請        承認済み 12件      │

│ ✅ ペア制約          設定済み 3件       │
│                                       │
│ ┌──────────────────────────────────┐ │
│ │    🔄 シフトを自動生成する          │ │
│ └──────────────────────────────────┘ │
│                                       │
```

- ボタンがローディングスピナーに変わる

- 完了後にシフト表画面へ遷移

## 8. アクセシビリティ

- WCAG 2.1 AA 準拠目標
- コントラスト比 4.5:1 以上
- キーボード操作対応（Tab / Enter / Arrow）
- シフト表のカラーには必ずテキスト略称を併記（色覚多様性対応）
